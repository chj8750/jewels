## 2. 서비스 요약
앱인토스 안에서 사용자가 돌을 깨고 랜덤 보석을 획득한 뒤, 보석함에 누적된 보석 가치가 1억원에 도달하면 토스포인트 1,000원으로 전환할 수 있는 광고 리워드형 미니게임 서비스다.

사용자는 매일 제공되는 돌을 깨며 보석을 모으고, 평균 20~30일 안에 1억원을 달성해 토스포인트 교환을 경험한다. 서비스의 1차 목표는 광고 매출 극대화보다 사용자의 재방문과 체류시간 증가다.

## 3. 목표

### 3.1 1차 목표
- 사용자의 재방문 빈도 증가
- 앱 내 체류시간 증가
- 매일 돌을 깨고 보석을 모으는 반복 루틴 형성
- 20~30일 주기로 토스포인트 교환 경험 제공

### 3.2 사업 목표
- 사용자가 1,000원 교환에 도달하기 전까지 사용자 1인당 광고 매출 3,000원 이상을 목표로 한다.
- 기본 돌 5개 기준 평균 약 24일 내 1억원 도달을 목표로 한다.
- 광고 eCPM과 실제 교환 속도에 따라 보석 가격, 확률, 광고 노출 조건을 운영 설정값으로 조정할 수 있어야 한다.

### 3.3 비목표
- 1차 버전에서는 강한 부정 탐지 시스템을 구축하지 않는다.
- 1차 버전에서는 계정당 교환 횟수 제한, 기기 제한, 비정상 클릭 패턴 탐지는 필수 범위에서 제외한다.
- 1차 버전에서는 복잡한 랭킹/친구 초대/소셜 경쟁 요소를 포함하지 않는다.

## 4. 사용자 경험 개요

### 4.1 핵심 사용자 시나리오
1. 사용자가 보석 발굴 서비스 첫 화면에 진입한다.
2. 첫 화면에서 오늘 깰 수 있는 돌 개수와 돌 이미지를 확인한다.
3. 사용자가 돌을 탭해 진행률을 올린다.
4. 진행률이 100%가 되면 돌이 깨지고 랜덤 보석이 등장한다.
5. 사용자가 `보석받기`를 누른다.
6. 전면광고가 노출된다.
7. 사용자가 전면광고를 끝까지 시청한다.
8. 광고 완료 후 보석이 보석함에 저장된다.
9. 보석함 누적 가치가 1억원에 도달하면 교환 유도 팝업이 노출된다.
10. 사용자가 `지금 교환하기` 또는 보석함의 `교환하기`를 누른다.
11. 토스포인트 교환 전 전면광고를 시청한다.
12. 광고 완료 후 토스포인트 1,000원 교환이 처리된다.
13. 교환 완료 기념으로 다이아 1개가 선물 지급된다.

### 4.2 주요 화면

#### A. 돌깨기 메인 화면
- 오늘 남은 돌 개수 표시
- 중앙 돌 이미지 표시
- 돌깨기 진행률 표시
- 가로 진행바 + 퍼센트 숫자 표시
- 진행바 위 망치 타격 애니메이션 표시
- 진행률에 따라 중앙 돌 이미지가 점점 깨지고 작아지는 애니메이션 표시

#### B. 보석 획득 결과 화면
- 획득한 보석 종류 표시
- 보석 가치 표시
- `보석받기` 버튼 표시
- `보석받기` 클릭 시 전면광고 노출

#### C. 보석함 화면
- 보유 보석 목록 표시
- 보석별 수량 표시
- 누적 보석 가치 표시
- 1억원까지 남은 금액 표시
- 1억원 이상이면 `교환하기` 버튼 활성화

#### D. 교환 팝업
- 1억원 달성 시 `지금 교환하기` 팝업 노출
- 토스포인트 1,000원 교환 가능 안내
- 교환 전 전면광고 시청 필요 안내

#### E. 교환 완료 화면
- 토스포인트 1,000원 교환 완료 안내
- 1억원 달성 기념 다이아 1개 선물 지급 안내
- 남은 보석 가치 표시

## 5. 기능 요구사항

### 5.1 돌 지급
- 사용자는 하루에 돌을 최대 5개까지 깰 수 있다.
- 하루 5개의 돌은 기본 제공된다.
- 하루 돌 개수는 운영 설정값으로 변경 가능해야 한다.
- 하루 기준은 서버 시간을 기준으로 한다.

### 5.2 돌깨기 인터랙션
- 사용자는 돌 이미지를 탭/클릭해 진행률을 채운다.
- 1회 탭당 진행률은 랜덤하게 5~15% 증가한다.
- 진행률은 최대 100%를 초과해 표시하지 않는다.
- 진행률이 100%가 되면 돌이 깨지고 랜덤 보석 결과가 노출된다.
- 돌깨기 중 앱을 나가거나 화면을 이탈했다가 다시 들어오면 진행률은 0%로 초기화된다.

### 5.3 보석 랜덤 지급
- 돌 1개를 100%까지 깨면 보석 1개가 랜덤으로 등장한다.
- 보석 종류는 다이아, 에메랄드, 사파이어, 루비, 금이다.
- 보석은 `보석받기` 후 광고 시청 완료 시점에 보석함에 저장된다.
- 광고를 중간에 닫거나 광고 완료 조건을 충족하지 못하면 보석은 지급하지 않는다.
- 광고 로딩 실패 시 보석은 지급하지 않는 것을 기본 정책으로 한다.

### 5.4 보석 가격 및 확률
| 보석 | 가치 | 등장 확률 |
|---|---:|---:|
| 다이아 | 400만원 | 5% |
| 에메랄드 | 250만원 | 8% |
| 사파이어 | 120만원 | 17% |
| 루비 | 50만원 | 30% |
| 금 | 20만원 | 40% |

- 보석 1개당 기대값은 약 83.4만원이다.
- 하루 5개 획득 기준 평균 약 24일에 1억원에 도달한다.
- 금/루비 같은 낮은 가치 보석이 주로 나오고, 다이아/에메랄드는 희귀 보상으로 설계한다.
- 실제 운영에서는 광고 eCPM과 교환 속도에 따라 가격과 확률을 운영 설정값으로 조정할 수 있어야 한다.

### 5.5 보석함
- 보석함에는 광고 시청 완료 후 획득한 보석만 저장된다.
- 보석함은 보석별 수량과 총 가치를 표시한다.
- 총 보석 가치가 1억원 이상이면 교환 가능 상태가 된다.
- 1억원 미만이면 교환 버튼은 비활성화하거나 부족 금액을 안내한다.

### 5.6 토스포인트 교환
- 교환 기준 금액은 총 보석 가치 1억원이다.
- 교환 리워드는 토스포인트 1,000원이다.
- 1억원 달성 시 `지금 교환하기` 팝업을 노출한다.
- 사용자는 보석함에서 직접 `교환하기`를 눌러 교환을 신청한다.
- 토스포인트로 바꾸기 시 전면광고를 시청해야 한다.
- 전면광고 시청 완료 후 교환을 처리한다.
- 교환 완료 후 누적 보석 가치에서 1억원만 차감한다.
- 1억원 초과분은 보석함에 유지한다.
- 교환에 사용된 보석만 소진하고, 사용되지 않은 보석은 유지한다.
- 교환 완료 시 1억원 달성 기념 보상으로 다이아 1개를 선물 지급한다.

### 5.7 광고 정책
- 돌을 깬 뒤 `보석받기` 버튼 클릭 시 전면광고를 노출한다.
- 토스포인트 교환 시에도 전면광고를 노출한다.
- 광고 완료 검증이 성공한 경우에만 보석 지급 또는 토스포인트 교환을 처리한다.
- 광고 노출 위치, 광고 노출 횟수, 광고 노출 조건은 운영 설정값으로 변경 가능해야 한다.

### 5.8 부정 참여 방지
- 1차 버전에서는 최소 정책만 적용한다.
- 하루 돌 개수 제한을 서버 기준으로 검증한다.
- 광고 완료 콜백/완료 상태를 검증한다.
- 광고 완료 검증이 실패하면 보석 지급 또는 교환을 처리하지 않는다.
- 계정당 교환 횟수 제한, 기기 제한, 비정상 클릭/반복 패턴 탐지는 추후 고도화 후보로 남긴다.

### 5.9 운영 설정
운영자는 아래 값을 변경할 수 있어야 한다.

| 설정값 | 설명 |
|---|---|
| 보석별 가격 | 다이아/에메랄드/사파이어/루비/금 가치 |
| 보석별 등장 확률 | 각 보석의 랜덤 등장 확률 |
| 하루 기본 돌 개수 | 기본값 5개 |
| 교환 기준 금액 | 기본값 1억원 |
| 리워드 금액 | 기본값 토스포인트 1,000원 |
| 광고 노출 위치 | 보석받기, 교환하기 등 |
| 광고 노출 횟수/조건 | 광고 시청 요구 조건 |
| 이벤트 보너스 | 이벤트 기간 추가 보상 |
| 기념 다이아 지급 여부 | 1억원 교환 완료 시 다이아 1개 지급 여부 |

## 6. 정책 및 밸런스

### 6.1 기본 밸런스
- 하루 기본 돌 개수: 5개
- 평균 도달 기간: 약 24일
- 목표 도달 범위: 20~30일
- 교환 기준: 보석 가치 1억원
- 교환 리워드: 토스포인트 1,000원

### 6.2 광고 수익 밸런스
- 목표: 사용자가 1,000원 교환에 도달하기 전까지 사용자 1인당 광고 매출 3,000원 이상
- 평균 20~30일 안에 1억원 도달 시 하루 5회 광고만 보면 총 광고 노출은 100~150회다.
- 이 경우 3,000원 매출 달성에 필요한 eCPM은 약 20,000~30,000원이다.
- 현재 추천 밸런스는 평균 약 24일 도달이며, 하루 5회 기준 총 120회 노출, 필요 eCPM은 약 25,000원이다.
- eCPM이 낮을 경우 광고 보상형 추가 돌/보너스/부스터를 도입해 하루 평균 광고 시청 수를 늘린다.

## 9. 예외 처리

### 9.1 광고 미완료
- 광고를 중간에 닫으면 보석 지급 또는 교환을 처리하지 않는다.
- 사용자에게 광고 시청 완료가 필요하다는 안내를 노출한다.

### 9.2 광고 로딩 실패
- 광고 로딩 실패 시 보석 지급 또는 교환을 처리하지 않는다.
- 사용자에게 잠시 후 다시 시도하라는 안내를 노출한다.

### 9.3 앱 이탈
- 돌깨기 중 화면을 이탈하면 진행률은 0%로 초기화된다.
- 이미 보석함에 저장된 보석은 유지된다.

### 9.4 교환 중 실패
- 교환 광고 완료 후 포인트 교환 처리에 실패하면 교환 상태를 재시도 가능 상태로 둔다.
- 보석 차감은 실제 토스포인트 교환 성공 이후 처리하는 것을 권장한다.

## 10. 개발 메모
- 보석 지급, 보석 가치 합산, 교환 차감은 서버 기준으로 처리하는 것을 권장한다.
- 클라이언트의 진행률, 애니메이션, 결과 노출은 사용자 경험용이며 최종 보상 지급은 서버 검증을 통과해야 한다.
- 보석 가격/확률/리워드/광고 조건은 하드코딩하지 않고 원격 설정 또는 운영 설정값으로 관리하는 것을 권장한다.
- 광고 완료 콜백은 중복 호출, 실패, 지연 가능성을 고려해 idempotent하게 처리해야 한다.
