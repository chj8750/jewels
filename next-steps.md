# 앱인토스 보석 발굴 서비스 다음 단계 실행 계획

## 결론
PRD는 1차 구현 논의가 가능한 수준으로 정리되었다. 다음 단계는 디자인 시안 → 개발 범위 산정 → 구현 → QA → 지표 확인 순서로 진행한다.

## 1. 바로 할 일

### 1.1 디자이너에게 전달
목표: PRD를 기준으로 화면 플로우와 UI 시안을 만든다.

필요 산출물:
- 돌깨기 메인 화면
- 보석 획득 결과 화면
- 보석함 화면
- 1억원 달성 팝업
- 토스포인트 교환 전 광고 안내 화면
- 교환 완료 화면
- 기념 다이아 지급 화면

확인 포인트:
- 돌깨기 진행률이 재미있게 보이는지
- 보석 획득 순간이 보상감 있게 보이는지
- 20~30일 목표가 사용자에게 너무 멀게 느껴지지 않는지
- 교환 전 전면광고 시청이 자연스럽게 이해되는지

## 2. 개발자에게 전달
목표: 1차 MVP 구현 범위와 기술 설계를 잡는다.

필요 산출물:
- 데이터 모델 초안
- API 목록
- 보석 지급 로직
- 광고 완료 검증 로직
- 교환 처리 로직
- 운영 설정값 구조
- 이벤트 로그 정의
- 1차 개발 일정 산정

핵심 구현 항목:
- 하루 돌 5개 제한
- 돌 탭 진행률 5~15% 랜덤 증가
- 100% 도달 시 보석 랜덤 결과 생성
- 보석받기 전면광고 완료 후 보석함 저장
- 보석함 누적 가치 계산
- 1억원 달성 팝업
- 교환 전 전면광고 완료 후 토스포인트 1,000원 처리
- 교환 후 1억원 차감 및 초과분 유지
- 기념 다이아 1개 지급
- 운영 설정값 관리

## 3. 마케터/기획 카피 정리
목표: 사용자에게 보이는 문구를 쉽게 정리한다.

필요 문구:
- 서비스 진입 문구
- 돌깨기 CTA
- 보석 획득 문구
- 광고 시청 안내 문구
- 1억원 달성 축하 문구
- 토스포인트 교환 문구
- 기념 다이아 지급 문구

## 4. MVP 검증 기준
1차 출시 전 최소 확인해야 할 지표:
- 사용자가 하루 돌 5개를 모두 깨는 비율
- 돌깨기 시작 후 100% 완료율
- 보석받기 광고 완료율
- 보석함 재방문율
- 7일 리텐션
- 평균 1억원 도달 예상 기간
- 교환 전 광고 완료율

## 5. 의사결정 필요 항목
아직 확정이 필요한 항목:
- 토스포인트 실제 지급 연동 방식
- 보석 차감의 상세 방식: 가치 기준 차감인지, 개별 보석 단위 차감인지
- 추가 광고 보상형 돌/부스터를 1차에 넣을지 여부
- 운영 설정값을 어디에서 관리할지
- 앱인토스 내 광고 SDK/전면광고 완료 콜백 검증 방식

## 6. 추천 진행 순서
1. 디자이너에게 PRD 전달 후 화면 플로우/시안 요청
2. 개발자에게 PRD 전달 후 MVP 기술 설계 요청
3. 시안과 기술 설계를 비교해 1차 범위 확정
4. 구현 시작 전 광고 SDK/토스포인트 연동 가능 여부 확인
5. MVP 구현
6. 내부 QA
7. 실제 광고 eCPM 기준으로 보석 가격/확률 재조정
8. 제한 배포 또는 테스트 운영
